Steps to Production

Phase 1: Pre-Launch Checklist (1-2 days)

  • Run database migrations - Execute supabase-new-features.sql in Supabase
  • Set up proper stock API - Replace Yahoo Finance with reliable provider
  • Configure email service - Set up Resend/SendGrid for password reset
  • Purchase domain - Buy and configure DNS
  • Set environment variables for Production in Vercel
  • Remove debug logging from API routes

Phase 2: Security Hardening (1-2 days)

  • Upgrade password hashing - Replace SHA-256 with bcrypt/argon2
  • Add rate limiting - Prevent API abuse
  • Set up CORS properly - Restrict origins
  • Add input sanitization - Prevent XSS/injection
  • Enable Supabase Row Level Security (RLS)
  • Set up SSL (automatic with Vercel)

Phase 3: Production Deployment (1 day)

  • Merge to main branch - Deploy to Production environment
  • Configure custom domain in Vercel
  • Set up DNS records (A, CNAME)
  • Verify SSL certificate is active
  • Test all critical flows (register, login, create portfolio, challenges)

Phase 4: Monitoring & Analytics (1 day)

  • Set up error tracking (Sentry)
  • Configure uptime monitoring (Uptime Robot)
  • Add analytics (PostHog, Vercel Analytics)
  • Set up alerts for errors and downtime

Phase 5: Legal & Compliance

  • Add Terms of Service page
  • Add Privacy Policy page
  • Add disclaimer - "Not financial advice, paper trading only"
  • Cookie consent banner (if required in your region)
